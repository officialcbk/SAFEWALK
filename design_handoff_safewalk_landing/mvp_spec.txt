Project Overview SafeWalk is a personal safety web application that monitors walking sessions and automatically alerts trusted contacts if something goes wrong. The web version is the first deliverable â€” it proves the product before moving to native mobile apps. The user opens the app, sets up trusted contacts, then taps Start Walk. The app tracks their location in real time, shows a check-in countdown, and escalates through a 5-stage ladder (prompt, SMS to contacts, voice call, 911 option, ongoing monitoring) if the user stops responding. A trusted contact who receives an SMS gets a time-limited web link showing the user&apos;s live location on a map â€” no login required. Key principle: This is a safety app. Every screen must load fast, work one-handed, and never feel confusing under stress. Clarity beats cleverness everywhere. 1.1 What &quot;shipped&quot; means for this MVP The MVP is complete when all of the following are true: A new user can sign up, verify their email, complete onboarding, add a trusted contact, start a walk, trigger SOS, and end the walk â€” end to end, no errors. All data (contacts, walk sessions, location history) is stored in Supabase, not localStorage. Row Level Security is enabled on every table so users can only see their own data. The trusted contact web view works on mobile browsers without login. Auth flows (sign up, sign in, reset password, email confirmation) all work correctly. The app passes Lighthouse accessibility audit with no critical failures. PIPEDA compliance controls are visible in Settings (data export, delete, privacy policy). 2. Each table below must be created with the columns and RLS policies described. Table: profiles Created automatically when a new user signs up via a Supabase trigger. Stores display name and phone number that are not part of auth.users. CREATE TABLE profiles ( id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY, full_name TEXT NOT NULL, phone TEXT, avatar_initials TEXT, -- e.g. &quot;AJ&quot; derived from full_name created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Trigger: create profile on signup CREATE OR REPLACE FUNCTION handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO profiles (id, full_name) VALUES (NEW.id, NEW.raw_user_meta_data-&gt;&gt;&apos;full_name&apos;); RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user(); -- RLS ALTER TABLE profiles ENABLE ROW LEVEL SECURITY; CREATE POLICY &quot;Users can view own profile&quot; ON profiles FOR SELECT USING (auth.uid() = id); CREATE POLICY &quot;Users can update own profile&quot; ON profiles FOR UPDATE USING (auth.uid() = id); Table: trusted_contacts Up to 5 trusted contacts per user. The is_primary flag designates the first contact who will be called (not just texted) during escalation Stage 3. CREATE TABLE trusted_contacts ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL, full_name TEXT NOT NULL, phone TEXT NOT NULL, email TEXT, is_primary BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ DEFAULT NOW() ); -- RLS ALTER TABLE trusted_contacts ENABLE ROW LEVEL SECURITY; CREATE POLICY &quot;Users manage own contacts&quot; ON trusted_contacts USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); Table: walk_sessions One row per walk. Status transitions: active -&gt; completed or active -&gt; sos_triggered. The share_token is a UUID used to build the trusted contact web view URL. CREATE TABLE walk_sessions ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL, status TEXT DEFAULT &apos;active&apos; CHECK (status IN (&apos;active&apos;, &apos;completed&apos;, &apos;sos_triggered&apos;, &apos;escalating&apos;)), destination TEXT, started_at TIMESTAMPTZ DEFAULT NOW(), ended_at TIMESTAMPTZ, duration_seconds INTEGER, distance_meters NUMERIC, share_token UUID DEFAULT gen_random_uuid() UNIQUE, share_expires_at TIMESTAMPTZ DEFAULT NOW() + INTERVAL &apos;24 hours&apos; ); -- RLS ALTER TABLE walk_sessions ENABLE ROW LEVEL SECURITY; CREATE POLICY &quot;Users manage own sessions&quot; ON walk_sessions USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); -- Public read via share_token (no auth required) CREATE POLICY &quot;Public read via share token&quot; ON walk_sessions FOR SELECT USING ( share_token IS NOT NULL AND share_expires_at &gt; NOW() ); Table: location_pings One row every 10 seconds during an active walk. This is what the trusted contact web view reads to show the live pin on the map. CREATE TABLE location_pings ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, session_id UUID REFERENCES walk_sessions(id) ON DELETE CASCADE NOT NULL, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL, lat NUMERIC(10, 7) NOT NULL, lng NUMERIC(10, 7) NOT NULL, accuracy NUMERIC, bearing NUMERIC, speed NUMERIC, recorded_at TIMESTAMPTZ DEFAULT NOW() ); -- Index for latest ping lookup CREATE INDEX ON location_pings (session_id, recorded_at DESC); -- RLS: owner can read/write, public can read if session share_token valid ALTER TABLE location_pings ENABLE ROW LEVEL SECURITY; CREATE POLICY &quot;Owner manages pings&quot; ON location_pings USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); CREATE POLICY &quot;Public read via active session&quot; ON location_pings FOR SELECT USING ( EXISTS ( SELECT 1 FROM walk_sessions ws WHERE ws.id = session_id AND ws.share_expires_at &gt; NOW() ) ); Table: escalation_events Audit log of every escalation action. This is used by the escalation ladder UI to show the user which stages have been triggered. CREATE TABLE escalation_events ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, session_id UUID REFERENCES walk_sessions(id) ON DELETE CASCADE NOT NULL, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL, stage INTEGER NOT NULL CHECK (stage BETWEEN 1 AND 5), event_type TEXT NOT NULL, -- &apos;check_in_missed&apos;, &apos;sms_sent&apos;, &apos;call_initiated&apos;, &apos;911_shown&apos; contact_id UUID REFERENCES trusted_contacts(id), resolved BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ DEFAULT NOW() ); ALTER TABLE escalation_events ENABLE ROW LEVEL SECURITY; CREATE POLICY &quot;Users manage own escalations&quot; ON escalation_events USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); 3.3 Supabase Auth configuration In the Supabase dashboard, configure the following before writing any auth code: Enable Email provider.
